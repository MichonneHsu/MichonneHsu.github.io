关于我的离职

##水深火热的深圳

去年7月份离职；同事和领导的冷嘲热讽，冷战、关系压制、负能量满满的。
刚入职的时候有位同事知道我效率很高，私底下找我说 能不能慢一点 晚上加班做完 就可以了
我寻思着你TMD能力有问题关我屁事？搞得好像动了他的蛋糕一样。
后来就没理他 紧接着就是冷嘲热讽地指责我这个没做好那个没做好，就当着同事和领导的面说我
我心想着， 你行 你厉害好吧。
这事情发酵了2个月 已经受不了了。
我发的工作报表部门领导从来不看，有一次我发了一个报表关于项目的进展，本来是砸规定的时间做完一个项目的
结果他连报表都不看一下，就以为我项目做完了很闲，就继续给我项目做，前一个还没做完就要我做后一个，
我一直都是给同行同事面子的，做的比较慢 也不能说比较慢 就是公司接的这个单搁置了很久都没做
要求员工加班做完 真的很难做 一边要给同行同事面子（就是那个一开始叫我项目做慢点的那个人）
一边还要尽快完成手头上的工作 后来我选择了综合 前面忘记说了 我是在这家公司做的是WEB前端开发工程师
17个页面的设计稿 我花了4天去完成 挨个抠图 量尺寸和像素而且还要适配不同的分辨率写各种CSS的媒体查询
我交稿的时候部门领导吐槽我说花了8天才做完 部门领导和我一样的年龄 我也是服了 这种人都能当领导？
我想知道他是怎么舔上去的。
4天前我就提交了工作报表就是死活不看，反正他看没看我是不知道的。
就这样 足足在那里干了4个月 公司在龙华区
从此以后我就认定了只要是在比较不繁荣的地区的公司都是很差劲的。

Swoole的协程知识点

关于协程的知识点

博主最近在学习swoole，学了一个星期;
有些要分享的小经验。
协程代码是必须写在go 函数里的

go(function(){
//to do
})

在此之前要说一下IO阻塞

什么是阻塞呢？
通俗地理解就是前方道路的阻碍。

go(function(){
echo 1;
co::sleep(1);
echo 2;
});

go(function(){
echo 3;

echo 4;

});

以上的代码执行顺序是 1 3 4 2！
要跨过这个河需要点时间是吧，但是呢你出门没有带任何的工具穿越这条河，你需要买一艘船，你买船需要时间是吧，这个时间就是co::sleep 第二个go协程是你要去买船的方法，当你买完后回来就会执行第一个go协程里的 echo 2;
要理解go协程需要知道 两个知识点：
1.代码是从上往下执行的。
2.代码的实行是需要时间的。

关于编程学习的事情-代码记不住的问题

代码记不住怎么办？

记不住代码是很正常的事情，就算是顶尖的程序员也会出现这种情况，同学的也别在意。也不是说你没有学编程的天赋；
不是每个人一生下来就会跑步，骑单车等等；其实做为一个程序员最重要的一件事情就是“解决问题的能力”，那么“解决问题的能力”取决于什么呢？
“解决问题的能力”取决于你的在编程的领域长期的经验累积，多敲代码吧骚年们，量变会产生质变。

文档api手册在手，天下我有！

程序员在做程序开发的时候必须打开编程手册，要养成多看手册的编程习惯，其实说白了手册就是你使用的语言的说明书；告诉你怎么用，有什么用，能改什么。
熟悉了就好，熟悉了之后就可以不用怎么翻手册了，反正就是忘记了就看手册，能记住最好，记不住没关系。

mysql 多集合检索技巧以及数据库查询优化小技巧

find_in_set

find in set 中文翻译就是在集合里找。

假设一个path字段里存放着 1,2,3 这种字符串类型的值
假设有三条数据他们的path字段里有 1,3 1,2 2,3

我在查询语句里写select find_in_set(3,path) from xxx;
查到的结果两个；分别是 1,3 和 2,3

说到数据库查询优化最好的办法就是让计算机能直接读懂代码，比如我要查时间 一般来说最好给时间戳去查找。
如果是多条件查询的话，在多个条件里第一位条件查询语句优先时间戳 select * from xxx where create_time = 10000000 …其他条件
这样的查询会比以往快很多，还有就是要给关键的字段附上key 一般来说 可以赋给 id create_time type name 等等。这样可以加快查询速度。
